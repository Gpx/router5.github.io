router5 is an HTML5 router, simple but powerful.

It favours convention over configuration, and is library / framework agnostic.
It plays nicely with modern component-based functional frameworks and libraries.
Router5 is made of several building blocks, each of them embrace the Single Responsability Principle.
  • path-parser PathParser is responsible for parsing paths into tokens (identifying segments, parameters, etc...) so it can match URL fragments or build them with supplied parameters.
  • route-node RouteNode organises routes in a tree of nested named routes with each of its branch being a valid route. It uses PathParser to quickly find a matching route in a tree, or to build a path from a route name and params.
  • router5 Finally, router5 is the last layer. It adds navigation and history functionalities, using RouteNode to store your application routes.
Helpers are being created for some specifi libraries and frameworks to help reduce boilerplate code.
  • router5-react In progress, currently including a Mixin and a Link component

  • router5-deku Will be added soon
  • Your favourite framework You would like router5 to integrate with your favourite framework or library? Open an issue and say hi!
